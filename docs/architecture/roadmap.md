# Roadmap

本文档用于整理 `v0.6.x` 之后的功能和结构推进方向，避免后续计划只散落在根 `README`。

## 当前阶段

当前仓库已经进入 `v0.6.0`：

- 编辑页支持 `单楼层` / `2.5D 总览` 双视图
- 2.5D 已接入轨道相机、全楼层播放和跨层额外过渡
- 电竞历史页已接入本地数据读取链路

接下来的工作重点应从“把功能做出来”，转向“把已有功能做完整、做稳定、做可扩展”。

## v0.6.x 重点

### 2.5D 总览补完

- 增加 `stairs / hatch` 的专用 2.5D 图元和楼层间分段连线
- 增加互动点路径高亮，明确当前播放实际经过的跨层节点
- 校准常用地图的 `overview_2_5d` 参数，减少楼层对齐偏差
- 完善 2.5D 下干员 icon、选中反馈和遮挡表达

### 电竞数据链路

- 为本地 JSON 和后续外部 API 建立统一的数据源抽象
- 补齐字段映射说明，明确 ban、阵容、赛事、队伍、比分等核心字段
- 控制页面展示字段，保持“看比赛优先”，避免页面退化成原始 JSON 浏览器

### 发布与维护

- 收口版本发布流程，统一 README、版本号、构建和 release note 的更新步骤
- 继续补打包与运行态校验，确保 `src/assets` 和 `data/esports` 在构建产物中可用

## v0.7.x 候选方向

- 评估电竞 / 复盘模式复用 `2.5D` 相机和跨层路径展示能力
- 评估导出能力，例如演示用截图、播放录制或回放分享
- 评估更完整的时间轴增强，如更细的关键帧组织、批量编辑、片段化回放

## 结构维护原则

- 新增功能优先落在现有分层目录中，不轻易再改顶层结构
- 文档优先描述边界和取舍，不堆无约束的功能愿望清单
- 新版本上线时，优先同步 `README`、版本号和对应策划 / 路线文档
