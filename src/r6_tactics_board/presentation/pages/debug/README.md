# Debug Page

这里放地图互动点、可破坏面与地图元数据调试页面。

## 当前文件

- `debug_page.py`: 地图 Debug 编辑页。

## 适合放这里的内容

- 调试页的表单、列表、交互反馈。
- 调试页与地图场景、资源索引之间的接线。
- 只在 Debug 工作流中使用的页面状态。

## 不要放这里的内容

- 互动点寻路算法。
- 地图资源扫描实现。
- 通用图元绘制代码。

## 当前支持的地图编辑对象

- `stairs`
  楼梯互动点，支持起点、终点与折线轨迹。
- `soft_wall`
  软墙战术面，以线段形式保存到地图资源。
- `hatch`
  Hatch 面，以面区域形式保存到地图资源，并承担跨层联通语义。

旧版 `hatch` 互动点语义已迁移到 `Hatch` 面，不应再新增 `hatch` 类型互动点。

## 落点规则

- 元数据编辑规则先看是否该进 `domain/` 或 `application/routing/`。
- 调试页只负责把这些规则暴露成 UI，不负责把算法重新写一遍。
