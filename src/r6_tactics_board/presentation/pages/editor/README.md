# Editor Page

这里放战术编辑页本身，以及只服务编辑页的本地模型。

## 当前文件

- `editor_page.py`: 主协调页，连接页面状态、应用层服务与图形场景。
- `editor_models.py`: 编辑页内部快照模型。

## 适合放这里的内容

- 编辑页级别的状态协调。
- 页面和 `application/`、`widgets/` 之间的接线。
- 只在编辑页内成立的本地快照模型。

## 不要继续堆到这里的内容

- 纯路径算法。
- 时间轴纯数据编辑。
- 可在别处复用的 Qt 控件。

## 落点规则

- 如果代码需要直接访问编辑页多个控件和页面状态，先考虑留在这里。
- 如果代码可以脱离页面独立测试，优先提到 `application/`。
- 如果代码主要是某块面板或某类控件，优先提到 `presentation/widgets/editor/`。
