# Pages

一级导航页面按业务域分目录，每个子目录负责一个完整页面及其页面内本地模型。

## 当前页面

- `editor/`: 主战术编辑页。
- `assets/`: 资源浏览页。
- `esports/`: 电竞历史记录浏览页。
- `debug/`: 地图互动点调试页。
- `settings/`: 设置页。

## 页面目录约定

- 一个页面一个子目录，页面主类命名为 `*_page.py`。
- 页面独有的本地快照、轻量模型、辅助函数，优先和页面放在同目录。
- 如果页面内控件开始被多个页面复用，再提取到 `presentation/widgets/`。

## 不要在页面目录里堆的内容

- 通用路径算法、通用时间轴数据逻辑。
- 资源扫描和工程持久化实现。
- 与页面无关的通用 Widget。

## 典型改动路径

- 新增一级页面：在这里建新子目录，然后去 `presentation/shell/main_window.py` 接入。
- 给编辑页新增一块侧栏：如果只在编辑页使用，先改 `editor/`；如果未来别的页面也用，再提取到 `widgets/editor/`。
- 给调试页新增互动点编辑规则：数据规则优先改 `application/`，页面表单和交互再落回这里。
