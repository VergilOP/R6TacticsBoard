# Presentation

表现层负责 Qt 界面，包括窗口、页面、场景、图元和复用控件。

## 子目录职责

- `shell/`: 应用外壳和导航入口。
- `pages/`: 一级页面和页面域内的本地模型。
- `widgets/`: 可复用图形部件和编辑器专用控件。

## 适合放这里的内容

- `QMainWindow`、`QWidget`、`QGraphicsScene`、`QGraphicsItem`。
- 布局、信号连接、快捷键、交互反馈。
- 页面层的状态协调和服务调用。

## 不要放这里的内容

- 资源扫描、工程持久化实现。
- 互动点寻路算法、关键帧纯数据变更。
- 脱离 Qt 也应该成立的业务规则。

## 落点规则

- 新增导航入口或页面编排，放 `shell/`。
- 新增一级页面，放 `pages/` 下独立子目录。
- 两个以上页面可能复用的 Qt 组件，放 `widgets/`。
- 只服务某个页面的局部 Widget，可以先放该页面子目录；一旦被复用，再提取到 `widgets/`。
