# Esports

这里负责读取 `data/esports/` 下的电竞历史数据，并把原始 JSON 转成项目内部可消费的结构。

## 适合放这里的内容

- `summary.json`、`raw_matches.json` 的路径定位和读取。
- 电竞数据的容错解析、字段归一化。
- 供页面或服务复用的数据加载入口。

## 不要放这里的内容

- `QWidget`、图标布局、列表渲染。
- 只为某个页面临时拼出来的展示文案。
- 编辑器业务逻辑或时间轴规则。

## 当前文件

- `esports_store.py`: 电竞数据目录定位、地图索引加载和比赛记录解析。

## 扩展约定

- 当前默认来源是本地 `json`，但内部输出已经统一到领域模型。
- 如果后面接入外部 API，优先复用同一套模型和归一化入口，不要让页面直接感知不同源格式。
