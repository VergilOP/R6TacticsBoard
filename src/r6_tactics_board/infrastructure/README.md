# Infrastructure

基础设施层负责“和外部怎么交互”，包括目录、文件、日志和运行环境适配。

## 子目录职责

- `assets/`: 资源根目录、默认目录、资源扫描与索引。
- `persistence/`: 工程文件读写。
- `diagnostics/`: 调试日志和运行期诊断。

## 适合放这里的内容

- 资源路径常量、目录查找、文件扫描。
- `*.r6tb.json` 的加载、保存和兼容处理。
- 调试开关、日志安装、环境检测。

## 不要放这里的内容

- 页面流程编排。
- 关键帧编辑规则、路径算法。
- `QWidget` 或页面交互文案。

## 依赖约定

- 可以依赖 `domain` 完成模型落盘和解析。
- 不应依赖 `presentation`。
- 新代码尽量保持“给定输入，返回结果”，不要在这里偷偷写页面逻辑。
