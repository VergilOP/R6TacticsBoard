# Domain

领域层只保留稳定业务概念，是其他目录共享的数据基础。

## 适合放这里的内容

- 地图、楼层、互动点、干员、关键帧、工程文件等核心模型。
- 稳定枚举、数据校验、序列化和反序列化辅助函数。
- 不依赖 Qt 的通用业务规则。

## 不要放这里的内容

- `QWidget`、`QGraphicsItem`、信号槽、样式、快捷键。
- 工程打开流程、资源扫描流程、页面选择状态。
- 只服务某个页面的临时视图模型。

## 当前文件

- `models.py`: 地图、干员、关键帧、工程等核心模型与解析函数。

## 落点规则

- 如果一个概念会被多个页面、多个服务同时使用，优先考虑先定义在这里。
- 如果一个结构只是编辑页内部快照，放到 `presentation/pages/editor/`。
- 如果一个逻辑主要是“如何操作这些模型”，放到 `application/`，不要继续堆进 `models.py`。
