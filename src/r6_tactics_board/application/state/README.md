# State

这里放可复用状态容器，重点是“多个场景都能复用的状态机制”。

## 适合放这里的内容

- 撤销 / 重做历史。
- 可独立测试的选择状态、缓存状态、快照栈。
- 不依赖具体页面 UI 的轻量状态对象。

## 不要放这里的内容

- 只在单个页面内部使用的字段集合。
- 需要直接访问 Qt 控件的状态。
- 资源扫描结果、工程持久化结果。

## 当前文件

- `history.py`: 通用撤销 / 重做历史栈。

## 落点规则

- 如果一个状态对象未来可能被编辑页、调试页或其他模块复用，优先放这里。
- 如果状态结构紧耦合某个页面的布局和控件，优先放页面目录本地维护。
