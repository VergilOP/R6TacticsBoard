# Application

应用层负责“如何做”。这里放纯编辑流程逻辑、控制器、会话服务和可复用状态，不放具体 Qt 界面拼装。

## 子目录职责

- `services/`: 组合资源、持久化等能力，对页面提供会话级 API。
- `routing/`: 跨楼层互动点寻路、路径拼接、插值计算。
- `timeline/`: 时间轴数据编辑控制器。
- `state/`: 可复用状态容器，例如撤销重做历史。
- `playback/`: 播放状态与播放控制。

## 适合放这里的内容

- “打开工程后要装配哪些数据”这类用例级逻辑。
- “一次编辑操作如何改动关键帧数据”这类纯逻辑控制器。
- 页面可调用、但不依赖页面本身的会话服务。
- 多个页面或控件都可能复用的状态容器。

## 不要放这里的内容

- `QWidget`、`QGraphicsScene`、信号连接、菜单和布局。
- 资源目录常量、文件选择对话框、直接读写 UI 控件文本。
- 只在某个页面内部短期存在的视图快照。

## 依赖约定

- 可以依赖 `domain`。
- 可以调用 `infrastructure` 的读写或资源能力。
- 不应依赖 `presentation`，避免反向耦合。

## 新文件落点

- 跨多个资源源或持久化入口的会话编排，放 `services/`。
- 地图互动点导航和楼层切换规则，放 `routing/`。
- 时间轴列、行、关键帧数据变更，放 `timeline/`。
- 通用撤销重做、选择集、缓存状态等，放 `state/`。
- 播放进度、暂停、速度等播放流程，放 `playback/`。
