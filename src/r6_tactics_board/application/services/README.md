# Services

这里放面向用例和会话的服务，职责是把多个底层能力组合成页面可以直接调用的 API。

## 适合放这里的内容

- 工程打开、保存、另存为、资源装配。
- 页面启动时需要的一次性会话初始化。
- 同时涉及资源索引和工程持久化的编排逻辑。

## 不要放这里的内容

- 单纯的数据结构定义。
- 纯数学计算、纯路径规划。
- 页面按钮点击后的提示框和消息文本。

## 当前文件

- `editor_session.py`: 工程读写、资源装配、页面会话级输入输出。

## 新文件落点

- 如果一个服务会同时调用 `asset_registry.py` 和 `project_store.py`，通常应该放这里。
- 如果一个逻辑没有外部依赖，只是操作内存中的关键帧数据，优先考虑 `timeline/` 或 `routing/`，不要放进服务层。
